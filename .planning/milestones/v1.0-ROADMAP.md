# Milestone v1.0: 井字棋 (Tic-Tac-Toe)

**Status:** ✅ SHIPPED 2026-06-17
**Phases:** 00-02
**Total Plans:** 6

## Overview

A complete, polished Tic-Tac-Toe desktop game with local two-player and AI opponent modes. Features smooth UI flow (MainMenu → Gameplay → GameOver), audio/music feedback, visual animation effects, and three-tier Minimax AI with alpha-beta pruning. Bilingual (zh_CN/en_US/ja_JP), ready to play.

## Phases

### Phase 00: Foundation, Sound & Localization

**Goal:** All infrastructure layers are stable and complete -- event system, audio, settings persistence, and bilingual support
**Depends on:** Nothing (brownfield, existing codebase)
**Plans:** 3 plans

Plans:
- [x] 00-01: Event system hardening (class_name on all events, StringName subscriptions, direct EventClass publishing), SettingsManager with ConfigFile persistence, SettingChangedEvent
- [x] 00-02: Audio system (AudioController autoload bridging EventBus→SoundManager, SoundRegistry, SFX/Music buses, kenney sounds, music crossfade), MiSans-Semibold CJK theme resource
- [x] 00-03: Bilingual localization (zh_CN/en_US JSON translation files, tr() adoption in all UI text, LanguageChangedEvent subscription, locale restore from SettingsManager)

**Details:**
- SoundManager registered only via plugin (not in project.godot [autoload]) — eliminates duplicate warnings
- AudioController creates SFX and Music buses at runtime if missing
- SettingsManager persists to `user://settings.cfg` via ConfigFile
- All game text uses tr() with translation keys; zero hardcoded Chinese strings
- I18NManager loads translations at startup, restores last-used locale from SettingsManager

### Phase 01: UI & Animations

**Goal:** The visual and interaction layer delivers polished menus, HUD, game flow, and animations
**Depends on:** Phase 0
**Plans:** 2 plans

Plans:
- [x] 01-01: Core game loop with UIManager (Board extraction as independent scene, MainMenu/HUD/GameOver UIPanel scenes, full UIManager migration replacing _show_only(), CQ-02 parameter validation, CQ-03 shared ui_helpers.gd, RematchEvent, ja_JP translation)
- [x] 01-02: Full polish (all animations: place mark bounce, win line draw, cell pulse, scene fades, button micro-interactions; PauseMenu with Esc GUIDE context + music volume reduction; Settings panel with 3 tabs Audio/Game/Language; second music track with crossfade switching)

**Details:**
- Board extracted as independent scene (`board.tscn`) with class_name Board
- All 5 UIPanels registered via UIRegistry and managed through UIManager stack
- Clean main.tscn: only Node2D root + CanvasLayer + TransitionOverlay
- GUIDE input contexts enabled/disabled per game state (board gameplay, pause menu)
- Win line draws across winning cells with TRANS_SPRING width 0→6 + alpha 0→1 fade
- Place mark bounce: TRANS_BOUNCE scale 0→1.2→1.0
- MiSans-Semibold global theme with font_size 20

### Phase 02: AI

**Goal:** Smart opponent with three-tier difficulty stands as the polished final challenge
**Depends on:** Phase 0
**Plans:** 1 plan

Plans:
- [x] 02-01: Full AI upgrade (alpha-beta minimax, three-tier difficulty Easy/Medium/Hard with differentiated parameters, randomized thinking delays, AiThinkingEvent, HUD thinking indicator with bilingual support, full test coverage)

**Details:**
- Easy: max_depth=1, random_chance=0.8, delay 0.5-1.0s
- Medium: max_depth=4, random_chance=0.15, delay 0.3-0.6s
- Hard: max_depth=9, random_chance=0.0 (perfect play, undefeatable), delay 0.2-0.4s
- AiThinkingEvent toggles HUD "AI thinking..." / "AI 思考中..." label
- call_deferred timer start prevents race condition with fast Hard difficulty
- Difficulty persists via SettingsManager ConfigFile

## Milestone Summary

**Key Decisions:**
- EventBus: Direct EventClass.new() over cached event type dictionary — simpler, less fragile, eliminates manual sync
- SoundManager: Registered only via plugin, not in project.godot [autoload] — avoids duplicate registration warnings
- Phase consolidation: Sound + Localization → Phase 0, UI + Animation → Phase 1, AI standalone → Phase 2
- Phase parallelism: Phase 1 and Phase 2 depend only on Phase 0, not on each other

**Issues Resolved (post-execution):**
- CR-01: Easy difficulty random_chance never applied (depth check short-circuited before random check) — FIXED
- WR-02: event.get() replaced with typed event.property access in hud.gd — FIXED
- WR-03: super._on_destroy() added to all 4 UIPanel subclasses — FIXED
- SND-02: Button click/hover sounds wired into UIHelpers.setup_button_animation() — FIXED
- Cursor loss on rematch: highlight.modulate.a reset + pulse tween killed in board._on_game_started — FIXED
- ESC broken on draw rematch: _esc_action moved outside is_connected guard — FIXED

**Technical Debt Incurred:**

- Phase 1: CONNECT_ONE_SHOT consumed when opening Settings from PauseMenu (mitigated by Resume button)
